# Quiet Current rebuild plan

## Phase 1 — simplify the product

- Preserve data, AI, import, privacy, and approval capabilities.
- Replace Flow/Proof/System with Ask/Activity/You.
- Remove technical and metaphorical language from primary paths.
- Establish progressive disclosure rules for confidence, sources, filters,
  diagnostics, and raw messages.

## Phase 2 — rebuild foundations

- Replace neon Flow Loom colors with the mineral Quiet Current palette.
- Rebuild theme, spacing, surfaces, buttons, fields, selection, navigation,
  sheets, empty/loading/error states, and accessibility behavior.
- Implement the paired current line as the only signature graphic.

## Phase 3 — rebuild the shell and journeys

1. Ask: brief, suggestions, conversation, composer, answer provenance,
   approval and undo.
2. Activity: month summary, search, review affordance, day groups, inspector,
   editing and import.
3. You: sources, intelligence, privacy, personalization and advanced tools.
4. Onboarding, app lock, consent, import progress and all failure states.

## Phase 4 — prove quality

- Analyze and test the complete project.
- Render phone, narrow phone, large text, dark mode, and tablet states.
- Audit copy, hierarchy, touch targets, contrast, overflow, idle frames, and
  screen-reader meaning.
- Remove all remaining Flow Loom vocabulary and visual primitives from live UI.

The rebuild is complete only when every user-facing state follows
`docs/design_system.md`; compiling is necessary but not sufficient.
